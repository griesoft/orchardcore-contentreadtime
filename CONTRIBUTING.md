# Contributing to Orchard Core Content Read Time Module

Thank you for your interest in contributing to this project! We welcome contributions from the community.

## How to Contribute

### Reporting Issues

If you find a bug or have a feature request:

1. Search existing issues to avoid duplicates
2. Create a new issue with a clear title and description
3. Include steps to reproduce (for bugs)
4. Specify your environment (.NET version, Orchard Core version)

### Submitting Pull Requests

1. **Fork the repository** and create your branch from `main`
2. **Make your changes** following the coding standards below
3. **Add or update tests** to cover your changes
4. **Ensure all tests pass** by running `dotnet test`
5. **Update documentation** if needed (README.md, XML comments)
6. **Commit your changes** with clear, descriptive commit messages
7. **Submit a pull request** with a description of your changes

## Development Setup

### Prerequisites

- .NET 8 SDK
- An IDE (Visual Studio, VS Code, or Rider)

### Building the Project

```bash
dotnet build
```

### Running Tests

```bash
dotnet test
```

## Coding Standards

- Follow the existing code style and conventions
- Use meaningful variable and method names
- Add XML documentation comments for public APIs
- Keep methods focused and concise
- Write unit tests for new functionality

## Code of Conduct

This project follows a [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## Questions?

If you have questions, feel free to open an issue for discussion.
